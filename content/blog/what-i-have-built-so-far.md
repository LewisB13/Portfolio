Over the last few days, I have been building out my personal portfolio website and turning it into more than just a static homepage.

So far, I have added:

- A personal homepage
- A blog section
- Project content
- Video/tutorial content
- GitHub Pages hosting
- A custom admin area using Sveltia CMS
- Markdown-based content
- Automatic blog loading from GitHub

One of the biggest improvements has been moving away from manually editing every post in JavaScript. Now I can create posts through the CMS, save them as Markdown, and have the site load them automatically.

This has helped me understand more about GitHub, static websites, content management, frontend structure, and deployment workflows.

The site is still a work in progress, but each improvement makes it easier to update and more useful as a proper portfolio.
