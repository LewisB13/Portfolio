One of the goals I set for myself this year was to build a personal portfolio website from scratch and host it publicly.

Rather than relying entirely on a drag-and-drop builder, I wanted to understand how a real deployment workflow works using GitHub Pages, custom domains, and content management tools.

This site started as a simple HTML/CSS project and gradually evolved into something much larger:

- custom styling
- blog functionality
- project pages
- video/tutorial sections
- GitHub deployment
- custom domain integration
- CMS-powered content editing

One of the biggest challenges was moving away from hardcoded content. Initially, every blog post and tutorial entry had to be manually edited directly in JavaScript files, which quickly became difficult to maintain.

To improve this, I integrated a Git-based CMS workflow so content can now be managed through an admin interface while still keeping the project fully hosted on GitHub Pages.

This setup gives me:

- version control
- free hosting
- editable content
- a public codebase for recruiters
- a better understanding of modern frontend workflows

I plan to continue improving the site by:

- adding more technical tutorials
- documenting projects and experiments
- improving responsiveness and UI design
- integrating project showcases
- writing about Linux, networking, cloud, and development topics

This portfolio is still a work in progress, but building it has already taught me a huge amount about deployment, frontend structure, Git workflows, and content management.
